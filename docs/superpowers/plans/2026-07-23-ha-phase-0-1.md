# HA Phase 0–1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship manual redundancy groups plus FortiGate FGCP discovery, exposing archetype-aware state in inventory and topology.

**Architecture:** SentinelNet currently stores managed devices in CSV/JSON and has no SQLAlchemy or tenant table. This slice adds a small SQLite-backed `redundancy` package, uses existing device `Group` as roadmap “tenant”, and enriches existing inventory/topology payloads without changing device records. Existing observability FortiGate polling calls FGCP discovery; no new scheduler or feature flag.

**Tech Stack:** Python 3.14, FastAPI, Pydantic v2, SQLite, requests, vanilla JavaScript, Vis.js, unittest, uv.

## Global Constraints

- Scope: roadmap Phase 0 plus Phase 1 only. Preview P1 and HA Phases 2–5 get separate plans.
- Preserve three archetypes: `ha_pair`, `stack`, `sso`; only `ha_pair` gets separate member nodes, heartbeat edge, checksum health, or standby routing.
- Map roadmap “tenant” to existing managed-device `Group`; preserve router RBAC through `user_group_scope`, `assert_group_allowed`, and `assert_device_allowed`.
- Use `core.data_config.get_path("redundancy.db")`; do not add ORM, SQLAlchemy, migrations framework, generic parser framework, feature flags, HA write controls, config hashing, ASA, SNMP, or standby routing.
- Store normalized member serials. Match discovery members to managed devices by normalized serial; leave device IP empty until matching device appears.
- Manual HA pairs require `virtual_ip`; manual stack/SSO groups require exactly one managed logical-device IP; duplicate logical-device bindings return HTTP 409.
- A failed FGCP request keeps last-known members and writes group health `unknown`; a one-member checksum response is `degraded`, not `out_of_sync`.
- Two active FGCP members set `split_brain` and write existing audit log event. Missing standby sets `degraded`; never delete group from transient discovery data.
- UI remains read-only for captured member data. VIP stays detail text, never map node.
- Every production behavior starts with a failing unittest. Use `uv run python -m unittest`; no pytest/lint task exists.

```python
class GroupType(str, enum.Enum):
    HA_PAIR = "ha_pair"
    STACK = "stack"
    SSO = "sso"

def parse_ha_status(ha_status, ha_checksums=None):
    res = ha_status.get("results", ha_status)
    raw_members = res.get("member") or res.get("statistics", {}).get("member", [])
    members = [MemberInfo(
        role=MemberRole.ACTIVE if (m.get("is_root_master") or m.get("is_root_primary")) else MemberRole.STANDBY,
        serial=m.get("serial_no") or m.get("serial"),
        model=m.get("model") or m.get("model_name"),
        firmware=m.get("version") or m.get("os_version"),
        mgmt_ip=m.get("mgmt_ip") or m.get("hostname_ip"),
        priority=_safe_int(m.get("priority")), state=MemberState.READY,
        details={k: m.get(k) for k in ("hostname", "sync_status", "uptime")},
    ) for m in raw_members]
    group = GroupInfo(group_type=GroupType.HA_PAIR,
        name=res.get("group_name") or res.get("cluster_id") or "FGCP-cluster",
        members=members, virtual_ip=res.get("cluster_virtual_ip") or res.get("vcluster_ip"),
        detection_source=DetectionSource.FORTIOS_API)
    checksums = ha_checksums.get("results", ha_checksums) if ha_checksums is not None else []
    checksums = checksums.get("member", []) if isinstance(checksums, dict) else checksums
    values = {row.get("checksum") for row in checksums if row.get("checksum")}
    group.health = GroupHealth.DEGRADED if ha_checksums is not None and len(checksums) < 2 else (GroupHealth.OUT_OF_SYNC if len(values) > 1 else GroupHealth.OK)
    group.health = _prefer_worse(group.health, group.compute_health())
    return group
```

Add `MemberRole` values `active`, `standby`, `master`, `member`, `unknown`; `MemberState` values `ready`, `down`, `version_mismatch`, `standby_hot`, `rp_down`, `provisioned`, `unknown`; `GroupHealth` values `ok`, `degraded`, `out_of_sync`, `split_brain`, `unknown`; and all six `DetectionSource` values listed in roadmap §1.1. Keep parser plain function; `CLI_PARSERS = {}` in this slice.

```javascript
function decorateRedundancyNode(node, r) {
  if (!r) return node;
  const colors = {active:'#2e7d32', standby:'#1565c0', out_of_sync:'#f9a825',
                  split_brain:'#c62828', degraded:'#f9a825', ok:'#455a64'};
  if (r.type === 'ha_pair') {
    const key = ['split_brain', 'out_of_sync'].includes(r.health) ? r.health : (r.role || 'ok');
    node.color = {border: colors[key] || colors.ok, background: node.color?.background || '#fff'};
    node.borderWidth = 3;
    node.label = `${node.label}\n[${key.replaceAll('_', '-')}]`;
  } else if (r.type === 'stack') {
    node.label = `${node.label}\n(stack ×${r.member_count ?? '?'})`;
    node.color = {border: r.health === 'degraded' ? colors.degraded : colors.ok};
    node.borderWidth = r.health === 'degraded' ? 3 : 1;
  } else if (r.type === 'sso') {
    node.label = `${node.label}\n[HA-SSO]`;
    node.color = {border: r.health === 'degraded' ? '#c62828' : colors.ok};
    node.borderWidth = r.health === 'degraded' ? 3 : 1;
  }
  return node;
}
```

Reuse `escapeHtml`, `apiFetch`, existing inventory panel classes, and existing Vis.js node/edge pipeline. A heartbeat edge uses `{ dashes: true, physics: false, label: 'HA' }`. Render HA pair member rows and B/C member drawers read-only. Manual CRUD form sends valid API payload; no priority edits, forced failover, or device-control action.
